# Template ‚Äî Next.js + Neon + Drizzle + Better Auth + R2

Project template with a layered architecture, documented conventions, Claude Code skills, and Vitest testing.

## Stack

- **Framework**: Next.js 16 (App Router)
- **Database**: Neon Postgres (serverless, scale-to-zero, branching) ‚Äî see [ADR-002](docs/adrs/ADR-002-database.md)
- **ORM**: Drizzle ‚Äî see [ADR-006](docs/adrs/ADR-006-orm.md)
- **Migrations**: Drizzle Kit ‚Äî see [ADR-007](docs/adrs/ADR-007-migrations-runner.md), [ADR-011](docs/adrs/ADR-011-migrations-on-deploy.md)
- **Auth**: Better Auth ‚Äî see [ADR-004](docs/adrs/ADR-004-authentication.md)
- **Email**: Resend ‚Äî see [ADR-009](docs/adrs/ADR-009-email-provider.md)
- **Storage**: Cloudflare R2 (public + private buckets) ‚Äî see [ADR-003](docs/adrs/ADR-003-storage.md), [ADR-010](docs/adrs/ADR-010-storage-upload-pattern.md)
- **Hosting**: Vercel ‚Äî see [ADR-001](docs/adrs/ADR-001-hosting.md)
- **Cron**: Vercel Cron ‚Äî see [ADR-005](docs/adrs/ADR-005-scheduled-jobs.md)
- **Styling**: Tailwind CSS 4
- **Validation**: Zod
- **Testing**: Vitest
- **Package Manager**: pnpm

## Usage

### 1. Copy the template

```bash
cp -r template/ ../my-new-project
cd ../my-new-project
```

### 2. Personalize

| File | What to change |
|------|----------------|
| `package.json` | project `name` |
| `src/config/app.config.ts` | `name`, domain limits |
| `src/config/routes.config.ts` | project-specific routes |
| `src/app/layout.tsx` | `metadata.title`, `metadata.description` |
| `src/app/page.tsx` | home page content |

### 3. Configure environment

```bash
cp .env.example .env.local
# Fill in the variables (DATABASE_URL, BETTER_AUTH_SECRET, RESEND_API_KEY, R2 keys, CRON_SECRET_KEY)
```

### 4. Install dependencies

```bash
pnpm install
```

### 5. Migrate and seed

```bash
pnpm db:generate    # generate SQL from the Drizzle schema (only when the schema changes)
pnpm db:migrate     # apply pending migrations
pnpm db:seed        # create a dev user via Better Auth
```

### 6. Develop

```bash
pnpm dev
pnpm test
pnpm test:watch
pnpm build          # runs db:migrate then next build (per ADR-011)
```

### 7. Create your first feature

Use the Claude Code skill:

```
/new-feature user-profile
```

This scaffolds: schema, entity, repository, service, action, validator, factory, and tests.

Each subfolder of `src/` has a `CLAUDE.md` with conventions. Claude Code reads them automatically.

## Global rules

- **`type` never `interface`** ‚Äî use intersections (`&`) for inheritance
- **Standalone functions** ‚Äî `export async function`, no factory pattern
- **Per-layer conventions** ‚Äî read the layer's `CLAUDE.md` before writing code in it

## Per-layer conventions

| Layer | Key rule |
|-------|----------|
| **Entities** | One folder per entity with `{name}.schema.ts` (Drizzle table) + `{name}.entity.ts` (`InferSelectModel` type) |
| **Repositories** | Try-catch, verbose logging with timing, `.limit(1)` + `[0] ?? null` for zero-or-one queries |
| **Services** | Transformers (no inline `.map()`), no direct DB access, authorization checks live here |
| **Actions** | `'use server'`, `getCurrentUser()`, try-catch, typed response, map `UnauthorizedError` |
| **Config** | Only `as const`, no functions |
| **Components** | Named exports, `'use client'` for hooks/events, actions as props |
| **Pages** | Orchestrator: `page.tsx` (server) + `page.client.tsx` (client) |
| **Tests** | Descriptive factories, mock repos with `vi.mock()` |

## Authorization

The database has no row-level security; authorization lives in the service layer. The pattern is:

1. The action reads the actor identity via `getCurrentUser()` and passes it to the service.
2. The service verifies ownership/role before any mutation or sensitive read.
3. If the actor is not allowed, the service throws `UnauthorizedError`.
4. The action catches it and maps it to a typed response.

## Storage uploads

Uploads bypass the Vercel function and go straight to R2 via a presigned URL. The flow:

1. Client calls `requestUploadUrl({ scope, mime, size })` from `src/actions/storage.actions.ts`.
2. The action validates MIME and size (defaults in `src/config/app.config.ts`), generates a presigned URL with `getSignedUrl`, and inserts a `storage` row.
3. The client `PUT`s the file directly to R2 using the presigned URL.

Two buckets are configured: `public` (direct URL) and `private` (signed URLs at read time).

## Cron jobs

One `route.ts` per cron under `src/app/api/cron/{name}/route.ts`. Each handler must call `assertCronAuth(request)` from `src/utils/cron.ts`. Schedules are declared in `vercel.json`.
